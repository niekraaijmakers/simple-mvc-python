Why This Structure?
===================

1. EXPLICIT DATABASE OPERATIONS
   - Uses db.session.query(Car) instead of Car.query
   - Shows exactly what's happening
   - SQL comments explain each operation
   - No "magic" to confuse beginners

2. LOGICAL GROUPING
   - model/ = All data-related code
   - controller/ = All HTTP handling
   - view/ = All presentation (HTML + CSS)

3. REPOSITORY PATTERN
   - Separates data access from HTTP
   - Business logic in one place
   - Easy to test and maintain

4. TEACHING FOCUSED
   - Clarity over brevity
   - Explicit over magic
   - Understanding over efficiency
